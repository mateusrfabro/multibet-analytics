# Project Name

This project is designed to manage and analyze user data through various SQL scripts and Python scripts. Below is a brief overview of the project's structure and its components.

## Project Structure

- **src/**: Contains the source code for the application.
  - **app.ts**: The main application file written in TypeScript.
  - **types/**: Directory for TypeScript type definitions.
    - **index.ts**: Type definitions used throughout the application.

- **sql/**: Contains SQL scripts for data management and analysis.
  - **dim_users_base.sql**: SQL statements for defining the base dimension for users, including table creation and initial data loading.
  - **tags/**: Directory for SQL logic related to user tags.
    - **PLAYER_NOT_VALID.sql**: Logic for identifying players that are not valid.
    - **POTENCIAL_ABUSER.sql**: Logic for identifying potential abusers based on certain criteria.
    - **NON_PROMO_PLAYER.sql**: Logic for identifying players who are not part of promotional activities.
    - **ENGAGED_AND_RG.sql**: Logic for identifying players who are engaged and have registered accounts.
    - **PLAYER_REENGAGED.sql**: Logic for identifying players who have been re-engaged after a period of inactivity.
    - **SLEEPER_LOW_PLAYER.sql**: Logic for identifying low-engagement players who are considered sleepers.
    - **BEHAV_RISK_PLAYER.sql**: Logic for identifying players who exhibit risky behavior.
    - **BEHAV_SLOTGAMER.sql**: Logic for identifying players who engage in slot gaming.
    - **VIP_WHALE_PLAYER.sql**: Logic for identifying high-value VIP players.
    - **WINBACK_HI_VAL_PLAYER.sql**: Logic for identifying high-value players that can be won back.

- **outputs/**: Directory for storing output files generated by the Python scripts.

- **export_smartico_risk.py**: Python script for exporting risk data related to Smartico, involving data extraction and transformation.

- **merge_mateus.py**: Python script for merging data related to a user or dataset named Mateus.

- **README_RUNBOOK.md**: Documentation for running the project, including instructions and details about the SQL scripts and Python scripts.

## Getting Started

To get started with this project, please refer to the `README_RUNBOOK.md` for detailed instructions on how to set up and run the application.
